# SkillMCP 使用注意事项

## ⚠️ 工具列表刷新机制

### 问题描述

当您打开一个新的技能包后，**AI 客户端可能不会立即看到新工具**。

**原因**：
1. MCP 客户端在连接时会缓存工具列表
2. 服务器端工具变化后，客户端不会自动刷新
3. 这是 MCP 协议的设计机制

### 解决方案

#### 方案 1：重新连接 MCP 服务器（推荐）

**Claude Desktop**：
```bash
# 1. 退出 Claude Desktop
# 2. 重新启动 Claude Desktop
```

**Windsurf**：
```bash
# 1. 设置 → MCP Servers
# 2. 禁用 SkillMCP
# 3. 重新启用 SkillMCP
```

**命令行测试**：
```bash
# 退出当前会话，重新运行
```

#### 方案 2：等待客户端自动刷新

部分 MCP 客户端支持定期刷新工具列表（如每 5 分钟），可以等待自动刷新。

### SkillMCP 的提示

打开技能包时，SkillMCP 会返回提示：

```json
{
  "success": true,
  "message": "✅ 技能包 'web' 已打开，可以使用 4 个新工具",
  "note": "💡 提示：如果看不到新工具，请重新连接 MCP 服务器"
}
```

### 未来优化

计划实现 MCP 的 `notifications/tools/list_changed` 通知机制，让服务器主动通知客户端工具列表变化，实现自动刷新。

---

## 📋 最佳实践

1. **打开技能包后**，检查返回消息中的提示
2. **如果看不到新工具**，重新连接 MCP 服务器
3. **完成任务后**，关闭不需要的技能包，减少工具数量
